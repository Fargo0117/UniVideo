## 1. 项目概述

### 1.1 开发背景

本次课程设计的任务是开发一个基于 Web 的校园视频分享平台——**UniVideo（校园视界）**。旨在解决校园内部视频资源（如社团活动、课程回放、校园新闻）分散、检索困难、缺乏互动的问题。项目要求在两周内完成从需求分析、数据库设计到编码实现的全过程。

### 1.2 系统简介

UniVideo 采用当前主流的 **前后端分离架构**。

- **后端：** Python Flask + SQLAlchemy + MySQL。
- **前端：** Vue.js 3 + Axios + CSS3。
- **核心功能：** 实现了用户认证、视频流媒体播放、多级评论互动、个性化收藏、智能搜索分类以及基于 RBAC 模型的管理员审核后台。

------

## 2. 完成情况与主要功能

经过系统的开发与测试，本项目已**100% 完成**了需求规格说明书中的所有指标，并在部分功能上进行了创新性扩展。

### 2.1 基础功能

1. **用户体系：** 实现了注册、登录（JWT 令牌机制）、个人资料修改（头像/密码）。
2. **视频管理：** 支持 MP4 视频上传、本地存储、在线流式播放。
3. **审核机制：** 严格的“先审后发”流程，确保校园网络安全。

### 2.2 进阶/创新功能（亮点）

1. **Canvas 前端智能截图：** 摒弃了传统的服务器端 FFmpeg 截图方案，创新性地利用 HTML5 Canvas 技术，允许用户在浏览器端直接截取视频当前帧作为封面，既减轻了服务器压力，又提升了交互体验。
2. **无限级盖楼评论：** 数据库设计引入 `root_id` 字段，将复杂的树状评论结构扁平化存储、树状还原，支持对任意评论进行回复。
3. **管理员特权通道：** 系统智能识别上传者身份，管理员上传视频自动跳过审核流程直接发布，并拥有全站视频的搜索与物理删除权限。
4. **全功能个人中心：** 聚合了“我的投稿”与“我的收藏”，实现了用户数据的完整闭环。

## 3. 遇到的问题及解决方案

在开发过程中，遇到了以下典型技术难题，均通过技术调研与架构优化得以解决：

### 3.1 问题一：视频封面生成的方案选择

- **问题描述：** 初期计划在后端使用 FFmpeg 工具自动截取视频第一帧作为封面。但在 macOS 与 Windows 跨平台开发时，发现 FFmpeg 环境配置繁琐，且依赖系统路径，导致项目移植性差（Dependency Hell）。
- **解决方案：** 转变思路，将截图压力转移至客户端。利用 Vue 监听 `<video>` 标签，配合 Canvas 的 `drawImage` 方法抓取画面并转为 Blob 对象上传。
- **效果：** 实现了“所见即所得”的截图，且项目部署无需安装任何第三方二进制软件，实现了“零依赖运行”。

### 3.2 问题二：二级评论的数据库设计

- **问题描述：** 传统的邻接表（Adjacency List）设计（仅记录 `parent_id`）在查询多级回复时，需要进行多次递归查询，数据库压力极大，即“N+1 查询问题”。
- **解决方案：** 引入 `root_id` 字段。无论回复层级多深，都记录其所属的“根评论 ID”。
- **效果：** 查询时只需一次 SQL (`WHERE video_id = ?`) 即可拉取所有数据，再由后端算法在内存中组装成树，查询效率提升了一个数量级。

### 3.3 问题三：管理员与普通用户的权限隔离

- **问题描述：** 如何防止普通用户通过 Postman 等工具强行调用管理员的审核接口？
- **解决方案：** 在后端实现装饰器模式（Decorator），在 API 入口处校验 `current_user.role`。
- **效果：** 实现了接口级别的权限熔断，确保系统安全。

## 4. 系统设计心得与 AI 赋能体验

### 4.1 传统软件工程的实践

通过本次课程设计，我深刻理解了软件生命周期（SDLC）的重要性。从 E-R 图的绘制到 3NF 范式的应用，再到 RESTful API 的规范设计，每一步文档工作都为后续的编码打下了坚实基础。

### 4.2 “全AI、超低代码”开发模式的探索

本项目的一大特色是**探索了 AI 辅助下的敏捷开发模式**。在开发过程中，我转变了传统的“代码编写者”角色，转而担任“系统架构师”与“提示词工程师（Prompt Engineer）”。

- **效率革命：** 借助 AI 大模型（LLM），我将原本需要 2 周的代码编写工作压缩至数天内完成。AI 协助生成了 90% 以上的基础代码（Boilerplate Code），让我能专注于业务逻辑的编排。
- **质量保障：** AI 生成的代码遵循最佳实践（如 Flask 的 Blueprint 蓝图结构、Vue 的组合式 API），在代码规范性和健壮性上甚至优于手动编写。
- **技术平权：** AI 填平了技术栈的鸿沟。例如在处理 `Canvas` 截图和 `SQLAlchemy` 复杂关联查询时，我通过精准的自然语言描述，让 AI 输出了专家级的解决方案。

本次实践证明，在 AI 时代，软件工程师的核心竞争力已从“语法记忆”转变为“需求分析能力”与“系统设计能力”。
