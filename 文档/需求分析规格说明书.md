## 1. 引言

### 1.1 编写目的

本文档详细描述了 **UniVideo（校园视界）** 系统的功能需求、性能需求、数据需求及运行环境。其编写目的是为了明确系统“做什么”，为后续的系统设计、编码实现、测试验收提供统一的标准和依据。预期的读者包括项目开发人员（李梓豪）、测试人员及课程设计指导教师。

### 1.2 项目背景

- **软件系统名称：** UniVideo 校园视界
- **任务提出者：** 软件工程课程设计指导教师
- **开发者：** 软件工程234班 李梓豪 (学号: 2318140407)
- **用户：** 大连交通大学全校师生、系统管理员
- **运行环境：** 校园局域网或单机演示环境
- **背景说明：** 随着多媒体技术的发展，校园内缺乏一个集中的视频资源分享与互动平台。本系统基于 Web 技术，旨在为学生提供一个可上传、分享、收藏、评论的流媒体社区，同时为管理员提供高效的内容监管工具。

### 1.3 定义

- **RBAC (Role-Based Access Control)：** 基于角色的访问控制，本系统分为普通用户（User）和管理员（Admin）两种角色。
- **Pre-audit（先审后发）：** 指视频上传后默认不可见，需管理员审核通过后方可发布的机制。
- **Nested Comment（盖楼评论）：** 指评论区支持多级回复，并以树状结构展示。

## 2. 任务概述

### 2.1 目标

本系统旨在构建一个功能完备的 B/S 架构视频分享平台，实现视频的云端存储（模拟）、在线播放及社区互动。重点解决校园视频资源分散、难以检索、缺乏互动的问题。

### 2.2 用户的特点

- **普通用户（学生）：** 数量较多，计算机水平参差不齐，要求系统界面简洁、操作直观（如拖拽上传、一键截图）。
- **管理员：** 数量较少，负责内容审核与系统维护，要求管理后台具备批量处理和快速检索能力。

### 2.3 假定和约束

- **开发周期：** 2周。
- **技术栈：** Python (Flask) + Vue.js 3 + MySQL。
- **存储约束：** 视频和图片文件存储于服务器本地物理磁盘，不依赖第三方云存储服务。

## 3. 需求规定

### 3.1 对功能的规定

本系统主要划分为 **前台用户子系统** 和 **后台管理子系统**，包含以下核心功能模块：

#### 3.1.1 用户认证模块

- **注册：** 用户使用学号、昵称和密码进行注册。
- **登录：** 系统验证账号密码，颁发 JWT Token，并根据角色（User/Admin）跳转至相应界面。
- **个人中心：**
  - **修改资料：** 用户可修改头像、昵称及登录密码。
  - **我的投稿：** 查看自己上传的所有视频及其审核状态。
  - **我的收藏：** 查看并管理自己收藏的视频列表。

#### 3.1.2 视频浏览与检索模块

- **分类导航：** 首页顶部提供“全部”、“校园生活”、“课程学习”等分类标签，点击可切换列表。
- **关键词搜索：** 支持通过视频标题模糊搜索视频。**系统强制过滤机制**保证搜索结果仅显示“已发布”状态的视频。
- **视频播放：** 点击视频卡片进入详情页，支持 HTML5 流媒体播放（播放/暂停/进度拖拽/音量调节）。

#### 3.1.3 视频投稿模块

- **文件上传：** 支持 MP4 格式视频上传。
- **智能封面：**
  - 方式一：本地上传图片文件。
  - 方式二：**在线截图**（利用 Canvas 技术），用户在预览器中拖动进度条，截取当前帧作为封面。
- **差异化发布策略：**
  - 普通用户上传：状态默认为 `0`（待审核）。
  - 管理员上传：状态自动为 `1`（已发布），实现免审绿色通道。

#### 3.1.4 互动社区模块

- **点赞：** 用户可对视频进行点赞/取消点赞，前端实时更新计数。
- **收藏：** 用户可将视频加入收藏夹，防止丢失。
- **评论系统：**
  - 发表一级评论。
  - 回复已有评论（二级评论），支持无限层级回复，前端以缩进或盖楼形式展示。

#### 3.1.5 后台管理模块

- **视频审核：** 管理员查看所有状态为 `0` 的视频，执行“通过”或“驳回”操作。
- **全站内容管理：**
  - **高级搜索：** 管理员可搜索全站任意状态的视频。
  - **删除视频：** 管理员具备最高权限，可物理删除违规视频及其关联文件。

### 3.2 对性能的规定

- **响应时间：** 页面加载及普通 API 请求响应时间不超过 1 秒；视频起播时间不超过 3 秒。
- **并发性：** 支持单机环境下的多用户并发访问（基于 Flask 多线程模式）。
- **存储容量：** 数据库支持万级记录存储，磁盘空间依赖于服务器硬件配置。

### 3.3 输入输出要求

- **输入：** 视频文件（MP4, <500MB）、图片文件（JPG/PNG, <5MB）、文本数据（UTF-8编码）。
- **输出：** 视频流数据、JSON 格式的 API 响应数据、错误提示信息。

### 3.4 数据管理能力要求

- **完整性：** 视频删除时，需级联删除相关的评论、点赞及收藏记录，防止脏数据。
- **一致性：** 收藏状态在详情页、列表页及个人中心需保持同步。

### 3.5 故障处理要求

- 当数据库连接失败或文件上传中断时，系统应向用户返回友好的错误提示（如“服务暂不可用，请稍后再试”），而不是显示代码报错信息。

## 4. 运行环境规定

### 4.1 设备

- **服务器：** 普通 PC 机或笔记本电脑（建议内存 8GB 以上，SSD 硬盘）。
- **客户端：** 任何安装有现代浏览器（Chrome, Edge, Firefox, Safari）的电脑或移动设备。

### 4.2 支持软件

- **操作系统：** Windows 10/11, macOS, 或 Linux (Ubuntu/CentOS)。
- **数据库管理系统：** MySQL 8.0 或以上版本。
- **后端运行环境：** Python 3.8+, Flask 2.0+。
- **前端运行环境：** Node.js 16+ (仅开发阶段), Nginx (生产部署)。

### 4.3 接口

- **数据通信协议：** HTTP/1.1 (RESTful API)。
- **数据交换格式：** JSON。

### 4.4 控制

- 系统通过 JWT (JSON Web Token) 进行会话控制，Token 过期需重新登录。
- 静态资源通过 Web 服务器的标准静态文件托管机制进行访问控制。
